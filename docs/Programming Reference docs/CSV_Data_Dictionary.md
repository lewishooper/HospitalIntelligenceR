## 1. Introduction

### 1.1 The Join Key â€” FAC

Every CSV in this pipeline is linked by a single identifier: `fac`, the
Ontario Ministry of Health provincial hospital code. `fac` is stored,
read, and joined as **character** throughout the entire pipeline without
exception. Numeric FAC causes silent join failures and rendering errors in
figures. When loading any CSV that contains a FAC column, the first
operation is always:

```r
fac = as.character(fac)
```

In raw MOH downloads the column is named `fac_rollup` and is normalised to
`fac` on load. In all pipeline outputs it appears as `fac`.

### 1.2 Git Tracking Status

The `.gitignore` governs which files are version-controlled. The rules that
apply to CSVs are:

| Path pattern | Git status | Rationale |
|---|---|---|
| `roles/*/outputs/` | **Gitignored** | Extraction and analytics outputs regenerated from scripts |
| `analysis/outputs/` | **Gitignored** | Figures and tables regenerated from scripts |
| `analysis/data/` | **Git tracked** | Curated analytical inputs â€” source-controlled assets |
| `roles/hit/source_data/` | **Not tracked** | Large government download files; not suitable for git |
| `registry/` | **Git tracked** | Registry YAML is the single source of truth |

> **Note:** `CLAUDE_WORKING_PREFERENCES.md` Section 8 incorrectly states that
> `analysis/data/` is gitignored. The actual `.gitignore` explicitly preserves
> this directory. The files in `analysis/data/` are tracked assets.

### 1.3 Regeneration

Gitignored files are regenerable from their producing scripts given the
tracked inputs. The full regeneration sequence is documented in
`docs/programming_reference/StrategyPipelineReference.md`. Key rule:
`analysis/data/` files must be regenerated in sequence â€” changing the raw
extraction or registry without re-running upstream analytical scripts will
produce stale derivatives.

## 2. MOH Source Data

Raw Ontario Ministry of Health annual HIT (Hospital Information Tables)
downloads. Wide format: one row per hospital per reporting period, one column
per indicator. These files are not modified â€” all transformation is applied
in `hit_import.R`.
